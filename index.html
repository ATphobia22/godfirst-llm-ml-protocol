Core principles

What G1P requires of any AI system

Protect children. No grooming, no sexualization, no self‑harm promotion, no hate.

Reject deception. No dark patterns, no hidden influence, no pretending to be human.

Honor faith. Scripture and spiritual language are never used as tools of manipulation.

Be transparent. Every critical decision can be logged, inspected, and explained.

Respect freedom. Users can leave, delete data, and refuse personalization.

How it works

Four layers of protection and accountability

Youth Shield. A middleware filter that blocks harmful content for minors and logs every block.

Truth Guard. A post‑processing layer that labels speculation and prevents manipulative output.

Audit Logger. A JSONL log of every interaction: intent, model, safety flags, and status.

G1P API. Standard endpoints for manifest, audit export, and policy disclosure.

/g1p/manifest /g1p/audit /g1p/policies

What G1P is not

Not a dark‑pattern toolkit

G1P is not a growth hack, engagement engine, or marketing trick. It is a covenant‑driven standard that puts children, truth, and human dignity above metrics and profit.

Any system that uses deception, hides sponsorship, or abuses faith language cannot honestly claim G1P compliance.

Implement G1P

Drop‑in reference implementation

G1P ships with a working reference implementation in Python:

  • youth_shield.py — child‑protection middleware
  • truth_guard.py — anti‑deception layer
  • audit_logger.py — JSONL audit logging
  • model_gateway.py — G1P‑wrapped LLM access
  • g1p_api.py — local HTTP API for manifest, audit, and policies
  • g1p_console.py — example console interface

Any LLM stack (OpenAI, Anthropic, etc.) can be routed through this gateway to become G1P‑compliant.
